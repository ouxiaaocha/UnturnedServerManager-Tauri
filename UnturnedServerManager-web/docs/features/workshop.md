创意工坊模组管理功能提供可视化界面管理服务器的 Steam Workshop 模组订阅，支持模组列表管理、备注功能、快速跳转 Steam 工作坊，以及模组缓存下载，让您轻松维护 `WorkshopDownloadConfig.json` 配置文件。

## 功能概览

| 功能 | 说明 |
| --- | --- |
| 模组列表管理 | 添加、删除、查看订阅的创意工坊模组 |
| 模组备注 | 为每个模组添加名称、用途、版本等备注信息 |
| Steam 工作坊跳转 | 一键打开浏览器访问模组的 Steam 工作坊页面 |
| 批量添加 | 支持逗号分隔批量添加多个模组 ID |
| 缓存下载 | 提前下载模组文件，减少服务器启动时间 |
| 更新监控 | 检测模组更新并自动重启服务器（可选） |

## 访问创意工坊模组管理

创意工坊模组管理功能整合在存档配置页面中：

1. 点击左侧导航栏的「存档」
2. 选择要配置模组的服务器存档
3. 切换到「工作坊」标签页

## 模组列表管理

### 查看模组

模组列表显示所有已订阅的创意工坊模组：
- **模组 ID**：Steam 创意工坊中的唯一标识符
- **模组备注**：您添加的自定义备注（名称、用途等）
- **工作坊链接**：点击链接图标快速跳转到 Steam 工作坊页面

### 添加模组

**单个添加**：
1. 在 Steam 创意工坊中找到目标模组
2. 复制模组 ID（URL 中的数字）
3. 点击「添加模组」按钮
4. 粘贴模组 ID
5. （可选）填写模组备注
6. 点击「确认」

**批量添加**：
1. 准备多个模组 ID，用逗号或换行符分隔
2. 点击「添加模组」按钮
3. 粘贴所有模组 ID（例如：`1234567890,2345678901,3456789012`）
4. 点击「确认」
5. 所有模组将一次性添加到列表

**如何获取模组 ID**：

Steam 创意工坊页面的 URL 格式为：
```
https://steamcommunity.com/sharedfiles/filedetails/?id=1234567890
```

其中 `1234567890` 就是模组 ID。

::: tip 提示
也可以右键点击工作坊页面的「订阅」按钮 → 「复制链接地址」，然后从链接中提取数字部分。
:::

### 删除模组

1. 在模组列表中找到要删除的模组
2. 点击该模组旁的「删除」按钮
3. 确认删除操作
4. 模组从订阅列表中移除

::: warning 注意
删除模组后，服务器需要重启才能使更改生效。已下载的模组文件不会自动删除。
:::

### 模组备注

为每个模组添加备注信息，方便管理和记忆：

**推荐备注内容**：
- **模组名称**：如「Elver 地图扩展」
- **模组用途**：如「添加新武器和载具」
- **版本信息**：如「v2.3.1」
- **依赖关系**：如「需要先安装 XXX 模组」
- **配置说明**：如「配置文件位于 XXX 目录」

**添加/编辑备注**：
1. 找到目标模组
2. 点击「编辑备注」按钮
3. 输入备注内容
4. 点击「保存」

备注信息保存在程序配置目录中，不会影响服务器的实际配置文件。

## Steam 工作坊跳转

快速访问模组的 Steam 工作坊页面：

1. 在模组列表中找到目标模组
2. 点击模组 ID 旁的**链接图标**（🔗）
3. 浏览器自动打开该模组的工作坊页面

**工作坊页面可以查看**：
- 模组详细介绍和说明
- 模组截图和视频
- 更新日志
- 用户评论和评分
- 模组作者信息

## 模组缓存管理

### 什么是模组缓存？

模组缓存是指将创意工坊模组文件提前下载到本地，而不是在服务器启动时才开始下载。

**缓存的优势**：
- ✅ 减少服务器启动时间（无需等待下载）
- ✅ 避免网络问题导致启动失败
- ✅ 提前验证模组完整性
- ✅ 支持离线环境启动服务器

### 启用模组缓存

在「工作坊」标签页的高级配置区域：
1. 勾选「启用模组缓存下载」
2. 配置缓存下载参数（可选）
3. 点击「保存配置」
4. 点击「立即下载缓存」按钮

缓存下载过程会显示进度，完成后可以查看下载结果。

### 查看缓存状态

模组列表会显示每个模组的缓存状态：
- ✅ **已缓存**：模组文件已下载到本地
- ⏳ **下载中**：正在下载模组文件
- ❌ **未缓存**：模组文件尚未下载
- ⚠️ **缓存过期**：本地缓存版本低于工作坊最新版本

### 更新缓存

当模组在 Steam 工作坊中更新后：
1. 缓存状态会显示「缓存过期」
2. 点击「更新缓存」按钮
3. 程序重新下载最新版本
4. 更新完成后重启服务器

### 清理缓存

如果需要清理过期或不再使用的模组缓存：
1. 点击「清理过期缓存」按钮
2. 程序扫描并列出可清理的缓存
3. 确认清理
4. 释放磁盘空间

## WorkshopDownloadConfig.json

创意工坊模组配置实际上是在编辑 `WorkshopDownloadConfig.json` 文件，该文件位于：

```
Servers/[存档名]/WorkshopDownloadConfig.json
```

### 文件格式

```json
{
  "File_IDs": [
    1234567890,
    2345678901,
    3456789012
  ],
  "Ignore_Children_File_IDs": [],
  "Query_Cache_Max_Age_Seconds": 600,
  "Max_Query_Retries": 2,
  "Use_Cached_Downloads": true,
  "Should_Monitor_Updates": false,
  "Shutdown_Update_Detected_Timer": 600,
  "Shutdown_Update_Detected_Message": "服务器将在 {0} 秒后关闭以更新模组"
}
```

### 配置项说明

| 配置项 | 说明 | 默认值 |
| --- | --- | --- |
| `File_IDs` | 订阅的模组 ID 列表 | `[]` |
| `Ignore_Children_File_IDs` | 忽略的子项模组 ID | `[]` |
| `Query_Cache_Max_Age_Seconds` | 查询缓存最大时长（秒） | `600` |
| `Max_Query_Retries` | 最大查询重试次数 | `2` |
| `Use_Cached_Downloads` | 使用缓存下载 | `true` |
| `Should_Monitor_Updates` | 监控模组更新 | `false` |
| `Shutdown_Update_Detected_Timer` | 检测到更新后关服倒计时（秒） | `600` |
| `Shutdown_Update_Detected_Message` | 关服倒计时提示消息 | 见上 |

### 手动编辑注意事项

如果选择手动编辑 JSON 文件：
- ✅ 保持 JSON 格式正确（注意逗号、引号、括号）
- ✅ 模组 ID 必须是纯数字，不要加引号
- ✅ 使用 UTF-8 编码保存
- ✅ 编辑前备份原文件
- ❌ 避免使用记事本编辑（可能破坏编码）

::: warning 注意
推荐使用创意工坊模组管理界面进行配置，避免因 JSON 格式错误导致服务器启动失败。
:::

## 高级配置

### 更新监控

启用后，程序会定期检查订阅模组是否有更新：
1. 勾选「启用更新监控」
2. 配置检查间隔（如每小时检查一次）
3. 配置关服倒计时（检测到更新后多久关闭服务器）
4. 配置关服提示消息

**工作流程**：
1. 程序定期查询 Steam 工作坊 API
2. 发现模组有更新
3. 在游戏内显示倒计时消息
4. 倒计时结束后关闭服务器
5. 下载更新后的模组
6. 自动重启服务器

::: tip 提示
更新监控适合需要保持模组最新版本的服务器。如果您希望手动控制更新时机，请关闭此功能。
:::

### 关服倒计时消息自定义

可以自定义检测到更新后的关服提示消息：
- `{0}` 占位符会被替换为剩余秒数
- 支持中文和英文
- 可以添加服务器相关信息

**示例消息**：
```
服务器将在 {0} 秒后重启以更新模组，请保存进度
Server will restart in {0} seconds for mod updates
模组更新中，{0} 秒后自动重启
```

### 忽略子项模组

某些大型模组包含多个子模组，如果您不想订阅某些子项：
1. 找到子项模组的 ID
2. 添加到「忽略子项 ID」列表
3. 保存配置

## 使用场景

### 场景 1：新服务器添加模组包

**需求**：新建服务器，需要添加一套常用模组（地图、武器、载具）。

**操作步骤**：
1. 在 Steam 工作坊收集模组 ID 列表
2. 复制所有 ID（用逗号分隔）
3. 在「工作坊」标签页点击「添加模组」
4. 批量粘贴所有 ID
5. 为每个模组添加备注
6. 启用模组缓存并立即下载
7. 启动服务器

**效果**：一次性完成所有模组的订阅和缓存，快速搭建模组服务器。

### 场景 2：模组版本管理

**需求**：某个模组更新后出现 Bug，需要回退到旧版本。

**操作步骤**：
1. 记录当前模组版本（在备注中）
2. 如果需要回退：
   - 从备份中恢复旧版本的模组文件
   - 或临时禁用「启用更新监控」
   - 或从工作坊下载旧版本（如果作者提供）

**提示**：建议定期备份 `Workshop/` 目录，以便需要时回退。

### 场景 3：测试新模组

**需求**：测试一个新模组是否与现有模组兼容。

**操作步骤**：
1. 在测试服务器（或本地）添加新模组
2. 启用缓存下载
3. 启动服务器测试
4. 如果兼容，在正式服务器添加
5. 如果不兼容，删除模组并记录备注

## 最佳实践

::: tip 最佳实践

1. **模组数量控制**：
   - 模组过多会增加服务器启动时间和内存占用
   - 建议控制在 20-50 个以内
   - 定期清理不再使用的模组

2. **备注管理**：
   - 为每个模组添加清晰的备注
   - 记录模组的依赖关系
   - 标注模组的用途和版本

3. **缓存策略**：
   - 正式服务器建议启用缓存下载
   - 定期检查和更新缓存
   - 清理过期缓存释放空间

4. **更新策略**：
   - 正式服务器谨慎使用自动更新监控
   - 建议在测试服务器先测试更新
   - 大版本更新前做好备份

5. **模组冲突排查**：
   - 新增模组前查看工作坊评论区
   - 记录已知的模组冲突组合
   - 遇到问题时逐个禁用排查

:::

## 常见问题

### 模组不生效怎么办？

排查步骤：
1. **检查模组 ID**：确认 ID 正确无误
2. **重启服务器**：模组配置需要重启才能生效
3. **查看服务器日志**：检查是否有模组加载错误
4. **验证模组兼容性**：确认模组支持当前 Unturned 版本
5. **检查模组依赖**：某些模组需要前置模组

### 如何查找模组 ID？

方法 1：从 URL 提取
```
https://steamcommunity.com/sharedfiles/filedetails/?id=1234567890
                                                        ↑ 这就是 ID
```

方法 2：在工作坊页面
1. 右键点击「订阅」按钮
2. 选择「复制链接地址」
3. 从链接中提取数字部分

方法 3：使用浏览器插件
某些浏览器插件可以自动提取和复制工作坊物品 ID。

### 模组下载失败怎么办？

可能原因和解决方法：
1. **网络问题**：检查网络连接，尝试使用加速器
2. **Steam API 限制**：稍后重试，或降低并发下载数
3. **模组已下架**：从订阅列表中删除该模组
4. **权限问题**：确保程序有写入 `Workshop/` 目录的权限

### 模组冲突如何排查？

逐个排除法：
1. 备份当前模组列表
2. 禁用一半模组
3. 启动服务器测试
4. 根据测试结果继续二分排查
5. 找到冲突的模组组合
6. 查看工作坊页面是否有兼容性说明

### 模组更新导致服务器崩溃怎么办？

1. **立即关闭服务器**
2. **禁用「更新监控」**
3. **从备份恢复旧版本模组文件**
4. **启动服务器验证稳定性**
5. **查看模组更新日志了解变更**
6. **在测试环境验证新版本**
7. **等待模组作者修复或回退**

### 如何批量管理多个服务器的模组？

如果您运行多个服务器：
1. 在第一个服务器配置好模组列表
2. 复制 `WorkshopDownloadConfig.json` 文件
3. 粘贴到其他服务器的存档目录
4. 在管理界面重新加载配置
5. 启动各服务器

或者使用共享模组缓存目录（需手动配置）。
