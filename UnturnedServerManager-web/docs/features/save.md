存档配置页面采用标签页布局，整合了服务器存档相关的所有配置功能，包括基础配置、插件管理、创意工坊模组、权限管理和 RCON 配置，让您在一个统一界面中完成所有存档设置。

## 功能概览

| 功能模块 | 说明 |
| --- | --- |
| 基础配置 | 服务器核心参数：地图、端口、玩家数、游戏规则 |
| 插件管理 | 查看和管理 Rocket 插件，添加插件备注 |
| 工作坊 | 管理 Steam 创意工坊模组订阅和配置 |
| 权限管理 | 编辑 Rocket 权限配置，管理权限组和玩家权限 |
| RCON 配置 | 配置 Rocket RCON 端口和密码 |

## 标签页导航

存档配置采用标签页布局，将原本分散的功能整合到一个页面中。点击顶部的标签可以在不同配置模块之间快速切换。

### 基础配置

管理服务器的核心参数和游戏规则。

#### 服务器信息

| 配置项 | 说明 | 示例 |
| --- | --- | --- |
| 服务器名称 | 在服务器列表中显示的名称 | `我的 Unturned 服务器` |
| 地图 | 服务器运行的地图 | `PEI`, `Washington`, `Russia` |
| 端口 | 服务器监听的端口 | `27015`, `27016` |
| 最大玩家数 | 允许同时在线的最大玩家数 | `24`, `48` |
| 服务器密码 | 连接密码（留空表示公开服务器） | `mypassword` 或留空 |

#### 游戏规则

| 配置项 | 说明 | 选项 |
| --- | --- | --- |
| PvE 模式 | 禁用玩家间伤害 | 启用 / 禁用 |
| 作弊模式 | 允许使用作弊命令 | 启用 / 禁用 |
| 视角限制 | 限制游戏视角 | 第一人称 / 第三人称 / 两者 |

#### GSLT 配置

GSLT（Game Server Login Token）是 Steam 游戏服务器登录令牌，用于将服务器注册到 Steam 服务器列表。

**如何获取 GSLT**：
1. 访问 [Steam 游戏服务器账户管理](https://steamcommunity.com/dev/managegameservers)
2. 登录 Steam 账号
3. 创建新的游戏服务器账户
4. 选择 **App ID: 304930**（Unturned）
5. 复制生成的令牌并粘贴到配置中

::: tip 提示
GSLT 令牌是可选的。如果不配置，服务器将只能通过 IP 地址直接连接，不会出现在 Steam 服务器列表中。
:::

#### Commands.dat 配置

Commands.dat 是服务器启动时自动执行的命令文件，可以配置欢迎消息、定时公告等。

**常用命令示例**：
```
say "欢迎来到我的服务器！"
time 12:00
weather none
```

在此标签页中可以：

### 工作坊

管理 Steam 创意工坊模组的订阅和配置。

主要功能：
- 添加/删除模组订阅
- 为模组添加备注
- 快速跳转到 Steam 工作坊查看模组详情
- 管理 WorkshopDownloadConfig.json

详细说明请参见 [创意工坊模组](/features/workshop) 页面。

### 权限管理

可视化编辑 Rocket 权限配置。

主要功能：
- 创建和编辑权限组
- 配置权限节点
- 通过 SteamID 分配玩家权限
- 配置权限继承关系

详细说明请参见 [权限管理](/features/permissions) 页面。

### RCON 配置

配置 Rocket RCON 远程管理功能。

| 配置项 | 说明 | 推荐值 |
| --- | --- | --- |
| RCON 端口 | RCON 监听端口 | `27115`（默认游戏端口 +100） |
| RCON 密码 | 远程连接密码 | 使用强随机密码 |

**功能**：
- 随机密码生成器：点击「生成随机密码」自动生成强密码
- 密码显示/隐藏：点击眼睛图标切换密码可见性
- 端口验证：自动验证端口范围是否合法

::: warning 安全提示
- RCON 密码应使用强随机密码，避免使用简单密码
- 不建议对公网开放 RCON 端口
- 定期更换 RCON 密码以提高安全性
:::

## 使用流程

### 新建服务器存档

1. 点击「新建存档」按钮
2. 输入存档名称（英文字母、数字、下划线）
3. 配置基础信息（地图、端口、最大玩家数）
4. 点击「创建」
5. 在其他标签页中完善配置（插件、模组、权限等）
6. 返回「服务器控制」页面启动服务器

### 修改现有存档配置

1. 从存档列表中选择要修改的存档
2. 切换到对应的配置标签页
3. 修改配置项
4. 点击「保存」按钮
5. 重启服务器使配置生效

### 配置优先级说明

当不同配置模块涉及相同设置时：
- **Config.txt 优先级最高**：Config.txt 编辑器中的配置会覆盖基础配置
- **Commands.dat 执行顺序**：按文件中的命令顺序依次执行
- **插件配置独立**：插件的配置文件独立于服务器配置

## 配置备份

### 自动备份

程序会在以下情况自动备份配置文件：
- 修改配置并保存时
- 服务器启动前
- 创建新存档时

备份文件位于：
```
backups/[存档名]/[时间戳]/
```

### 手动备份

建议定期手动备份重要配置：
1. 复制整个存档目录：
   ```
   Servers/[存档名]/
   ```
2. 保存到安全位置（如云盘、移动硬盘）

### 恢复备份

1. 进入 `backups/[存档名]/` 目录
2. 找到要恢复的备份版本
3. 将备份文件复制回 `Servers/[存档名]/` 目录
4. 覆盖现有文件
5. 重启服务器

## 常见问题

### 如何在标签页之间切换？

点击页面顶部的标签名称即可切换。所有标签页共享同一个存档选择器，切换标签不会改变当前选中的存档。

### 修改配置后需要重启服务器吗？

大部分配置修改需要重启服务器才能生效：
- ✅ **需要重启**：地图、端口、游戏规则、RCON 端口
- ⚠️ **部分需要重启**：插件配置（取决于插件）
- ❌ **不需要重启**：插件备注、模组备注

建议修改配置后重启服务器以确保生效。

### 标签页布局的优势是什么？

相比分散的独立页面，标签页布局提供：
- **统一入口**：所有存档相关配置集中在一个页面
- **快速切换**：无需重新选择存档，标签间切换更流畅
- **逻辑清晰**：按功能模块分类，配置结构一目了然
- **操作高效**：减少页面跳转，提升配置效率

### 如何知道哪个标签页需要配置？

基本配置流程：
1. **基础配置**（必需）：服务器名称、地图、端口等核心参数
2. **RCON 配置**（推荐）：配置远程管理
3. **工作坊**（可选）：如果需要使用模组
4. **插件管理**（可选）：如果安装了插件
5. **权限管理**（推荐）：配置管理员和玩家权限
