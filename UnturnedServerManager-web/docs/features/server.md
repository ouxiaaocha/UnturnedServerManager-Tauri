# 服务器控制

服务器控制页面提供对 Unturned 服务器的完整控制能力，包括启动、停止、重启、本地命令发送和实时日志查看。

## 核心功能

### 服务器操作

| 操作 | 说明 |
| --- | --- |
| 启动 | 启动 Unturned 服务器进程 |
| 停止 | 通过本地命令 Bridge 发送保存和关闭命令，等待服务器保存数据后退出 |
| 重启 | 先通过本地命令优雅关闭，等待旧进程退出后再启动 |
| 强制停止 | 立即终止服务器进程，不等待数据保存 |

### 本地命令

服务器运行时，可以在输出区下方输入 Unturned 控制台命令，例如：

```text
Save
Say hello
Shutdown
```

命令会写入当前存档目录中的本地队列文件，再由 `UnturnedServerManagerBridge.dll` 在服务器进程内读取并执行。这个路径不走 RCON，不监听公网端口，也不会把命令写入操作审计日志。

### 实时输出

服务器控制页面实时显示服务器的控制台输出：

- 支持自动滚动到最新日志
- 支持日志搜索功能，快速定位关键信息
- 支持清空当前显示的日志
- 本地命令发送成功后立即追加 `[命令] > ...` 记录

### 运行模式

支持两种服务器运行模式：

- **局域网模式**：仅局域网内的玩家可以连接
- **互联网模式**：公网玩家可以连接（需要正确配置端口转发）

::: warning 注意
切换运行模式需要重启服务器才能生效。
:::

## 使用场景

### 日常管理

- 查看服务器运行状态
- 启动/停止服务器
- 发送 `Save`、`Say` 等本地控制台命令
- 查看实时日志输出

## 运行条件

本地命令功能依赖 Rocket.Unturned 和内置 Bridge 插件。软件启动和服务器启动前会自动检查，设置页面也可以手动检测或重新部署。

### 故障排查

- 通过日志搜索定位错误信息
- 查看服务器启动过程中的警告和错误
- 强制停止卡死的服务器进程

### 模式切换

- 在局域网测试和公网部署之间切换
- 根据需要调整服务器运行模式
