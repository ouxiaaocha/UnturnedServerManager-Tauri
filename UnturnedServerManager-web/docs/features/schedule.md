# 定时任务

定时任务页面用于创建和管理服务器的自动重启任务，保障服务器稳定运行。

  <img src="/schedule.png" width="90%" alt="定时任务界面" />
</p>

## 任务类型

### 每日任务

在每天的指定时间执行：

- 适合日常维护重启
- 可设置具体时间（如每天凌晨 4:00）

### 每周任务

在每周的指定日期和时间执行：

- 适合周常维护
- 可选择星期几执行

### 间隔任务

每隔指定时间周期执行：

- 适合高负载服务器
- 可设置间隔小时数（如每 6 小时）

## 任务配置

### 基本设置

| 配置项 | 说明 |
| --- | --- |
| 任务名称 | 任务的显示名称 |
| 任务类型 | 每日/每周/间隔 |
| 执行时间 | 具体执行时间或间隔 |
| 启用状态 | 是否启用该任务 |

### 提前提醒

支持在重启前发送提醒消息：

- 提前 N 分钟提醒玩家
- 自定义提醒消息内容
- 通过 RCON 广播提醒

::: tip 提示
建议在重启前提醒玩家，避免玩家因突然重启而丢失游戏进度。
:::

## 使用场景

### 日常维护

设置每日凌晨自动重启，清理服务器内存，保持最佳性能。

### 定期更新

配合服务端更新功能，在更新后自动重启服务器。

### 高负载管理

对于高玩家数量的服务器，设置间隔重启以防止内存泄漏。

## 注意事项

::: warning 数据保存
自动重启前会发送提醒消息，但请确保服务器配置了自动保存功能，避免玩家数据丢失。
:::

::: danger 任务冲突
避免设置过于频繁的重启任务，以免影响玩家游戏体验。建议间隔至少 4 小时以上。
:::
